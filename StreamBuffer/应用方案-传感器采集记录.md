# 传感器采集记录方案（StreamBuffer + ThreadQueue 三库 + 中值滤波）

> **场景**：100Hz 传感器，每条 `1024*3` 个 int（12KB）；原始数据记录 bin；每条提取 `3*9=27` 关键点；10 条滑动窗口中值滤波（满丢最老）输出连贯稳定值；过程中接收外部控制指令一条一条执行。
> **平台**：IMX6ULL (ARM Linux)  | **作者**：zlzksrl  | **日期**：2026-07-09
> **用到的库**：`StreamBuffer`（写盘）、`ThreadQueue`（指令）、中值滤波（直接数组最省；`WindowQueue` 为可选扩展）

---

## 一、场景与需求

| 编号 | 需求 | 说明 |
|------|------|------|
| **A** | 原始数据 → bin | 100Hz × 12KB 原始帧持续记录到二进制文件 |
| **B** | 提取关键点 | 每条原始数据提取 3×9=27 个关键值 |
| **C** | 滑动中值滤波 | 维护 10 条 × 27 值的窗口，满 10 丢最老，对 27 通道各取中值，输出连贯稳定 |
| **D** | 控制指令执行 | 外部指令一条一条顺序执行，不能丢 |

---

## 二、数据量分析

| 数据 | 频率 | 单条大小 | 速率 |
|------|------|---------|------|
| 原始帧 | 100Hz | 1024×3 int = 12KB | **1.2 MB/s** |
| 关键点 | 100Hz | 27 int = 108B | 10.8 KB/s |
| 中值滤波计算 | 100Hz | 27 通道 × 10 窗口 | ~12 万次比较/s（< 0.1% CPU） |

→ 原始数据量大需**攒批写盘**；关键点/滤波计算量极小。

---

## 三、数据流架构

```
                         ┌─────────────────────────────┐
传感器 100Hz (12KB/帧) ──→│ 采集线程                    │
                         │  ① PutData(12KB原始)         │
                         │  ② 提取 27 关键点            │
                         │  ③ 中值滤波(直接数组10×27)   │── 27 个滤波值(连贯稳定)
                         └──────┬──────────────────────┘
                                │ ①
                                ▼
                    ┌──────────────────────┐
                    │     StreamBuffer      │  攒批、零拷贝
                    │   (环形字节流缓冲)    │
                    └──────────┬───────────┘
                                │ GetDataAddress
                                ▼
                         ┌──────────────┐
                         │  写盘线程     │── fwrite *.bin
                         └──────────────┘

  外部指令源 ──PutMsg──→ 【ThreadQueue】 ──GetMsg──→ 指令执行线程（一条一条）
```

---

## 四、三库分工

| 库 | 角色 | 关键特性 |
|----|------|---------|
| **StreamBuffer** | 原始 12KB → bin 文件 | 攒批写盘减 I/O、`GetDataAddress` 零拷贝、生产不阻塞（满则丢新） |
| **中值滤波（直接数组）** | 27 值 × 10 条滑窗 | 环形覆盖最老 O(1)、插入排序、栈数组零 malloc、单线程无锁 |
| **ThreadQueue** | 控制指令 FIFO | 一条一条消费、满则阻塞（指令绝不能丢） |
| WindowQueue（可选） | 多线程共享窗口时用 | 线程安全窗口访问 + 满丢最老（见第八节） |

---

## 五、中值滤波（消耗最低实现）

对 27 个通道，各自取窗口内 10 个采样的中值。固定 10×27 小规模，**直接用数组环形缓冲最省**（无需队列开销）：

```c
#include <string.h>

#define NCH 27      /* 3*9 通道 */
#define WIN 10      /* 窗口大小 */

static int win[NCH][WIN];   /* 每通道 10 值，环形复用，零 malloc */
static int wpos = 0;        /* 窗口写入位置 */

/**
 * @brief  滑动中值滤波：输入 27 个新关键点，输出 27 个滤波值
 * @note   环形覆盖最老(O(1)) + 插入排序(10元最快) + 栈数组(无malloc) + 单线程无锁
 */
void filter_median(const int kp[NCH], int out[NCH])
{
    int c, i, j;
    for(c = 0; c < NCH; c++)
    {
        int tmp[WIN];
        win[c][wpos] = kp[c];                  /* 覆盖最老(环形) */
        memcpy(tmp, win[c], sizeof(tmp));      /* 复制到栈上排序(不动原窗口) */
        for(i = 1; i < WIN; i++)               /* 插入排序(10 元最快) */
        {
            int k = tmp[i];
            for(j = i - 1; j >= 0 && tmp[j] > k; j--) tmp[j + 1] = tmp[j];
            tmp[j + 1] = k;
        }
        out[c] = tmp[WIN / 2];                 /* 中值 = 第 5 个 */
    }
    wpos = (wpos + 1) % WIN;                   /* 环形推进 */
}
```

**为什么最省**：① 环形覆盖最老，O(1) 不搬移；② 插入排序（10 元，比 `qsort` 函数调用开销小）；③ 栈数组无 malloc；④ 单线程无锁。消耗 ≈ 27×45 ≈ 1200 次/条 × 100Hz ≈ 12 万次/s，**< 0.1% CPU**。

## 六、完整 Demo（三库串联）

> 下面 demo 把采集→写盘→滤波→指令串起来，模拟传感器（`gen_sensor`）、关键点提取（`extract_kp`）用占位实现，重点演示**三库如何协同**。编译需链接 `StreamBuffer` 与 `ThreadQueue` 两个库（滤波是内联函数，不用 WindowQueue）。

```c
/**
 * @file sensor_demo.c
 * @brief 100Hz 传感器采集：原始写 bin + 关键点中值滤波 + 指令执行
 */
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <pthread.h>

#include "StreamBuffer.h"      /* 原始数据写盘 */
#include "ThreadQueue.h"       /* 控制指令 */

#define RAW_INT   (1024 * 3)   /* 原始帧: 1024*3 int = 12KB */
#define NCH       27           /* 3*9 关键点通道 */
#define WIN       10           /* 中值滤波窗口 */
#define FRAMES    500          /* 本 demo 采集帧数 */

/* ========== 中值滤波（直接数组，最省） ========== */
static int g_win[NCH][WIN];
static int g_wpos = 0;

static void filter_median(const int kp[NCH], int out[NCH])
{
    int c, i, j;
    for(c = 0; c < NCH; c++)
    {
        int tmp[WIN];
        g_win[c][g_wpos] = kp[c];
        memcpy(tmp, g_win[c], sizeof(tmp));
        for(i = 1; i < WIN; i++){ int k=tmp[i]; for(j=i-1;j>=0&&tmp[j]>k;j--) tmp[j+1]=tmp[j]; tmp[j+1]=k; }
        out[c] = tmp[WIN / 2];
    }
    g_wpos = (g_wpos + 1) % WIN;
}

/* ========== 共享上下文 ========== */
typedef struct {
    T_StreamBuffer   *sb;      /* 原始数据缓冲(写盘) */
    T_ThreadQueueMsg *tq;      /* 指令队列 */
    int    acq_done;           /* 采集结束标志 */
} Ctx;

/* 模拟：生成一帧原始数据 */
static void gen_sensor(int *frame, int idx)
{
    int i;
    for(i = 0; i < RAW_INT; i++) frame[i] = idx + i;   /* 占位：实际接传感器 */
}

/* 模拟：从原始帧提取 27 关键点 */
static void extract_kp(const int *frame, int kp[NCH])
{
    int i;
    for(i = 0; i < NCH; i++) kp[i] = frame[i * (RAW_INT / NCH)];   /* 占位：实际按算法 */
}

/* ========== 采集线程：原始入 StreamBuffer + 提取关键点 + 滤波 ========== */
static void *acq_thread(void *arg)
{
    Ctx *ctx = (Ctx *)arg;
    int frame[RAW_INT];        /* 12KB 原始(栈) */
    int kp[NCH], out[NCH];
    int idx;
    for(idx = 0; idx < FRAMES; idx++)
    {
        gen_sensor(frame, idx);
        /* ① 原始 12KB 入 StreamBuffer(写盘) —— 不阻塞,满则丢新 */
        StreamBufferAPI_PutData(ctx->sb, (const char *)frame, sizeof(frame));
        /* ② 提取关键点 */
        extract_kp(frame, kp);
        /* ③ 中值滤波 → out 即连贯稳定输出 */
        filter_median(kp, out);
        /* 此处可把 out 上报/记录(本 demo 仅打印前几帧) */
        if(idx % 100 == 0)
            printf("[acq] frame=%d kp0=%d filtered0=%d\n", idx, kp[0], out[0]);
        usleep(10 * 1000);     /* 100Hz = 10ms */
    }
    ctx->acq_done = 1;
    return NULL;
}

/* ========== 写盘线程：StreamBuffer → bin(零拷贝) ========== */
static void *dump_thread(void *arg)
{
    Ctx *ctx = (Ctx *)arg;
    FILE *fp = fopen("sensor.bin", "wb");
    char *ptr;
    int   used;
    if(fp == NULL) { printf("fopen bin fail\n"); return NULL; }
    while(1)
    {
        int r = StreamBufferAPI_Wait(ctx->sb, 1000, &used);   /* 等阈值/超时/关闭 */
        if(r > 0)
        {
            int n;
            /* GetDataAddress 零拷贝：直接 fwrite 内部地址，避免 12KB 再拷一次 */
            while((n = StreamBufferAPI_GetDataAddress(ctx->sb, &ptr, 65536)) > 0)
                fwrite(ptr, 1, (size_t)n, fp);
        }
        if(r <= -2) break;       /* CLOSE_EMPTY / 错误 → 退出 */
    }
    fclose(fp);
    printf("[dump] bin closed\n");
    return NULL;
}

/* ========== 指令线程：PutMsg 生产(模拟外部指令源) ========== */
static void cmd_release(void *data){ free(data); }

static void *cmd_prod_thread(void *arg)
{
    Ctx *ctx = (Ctx *)arg;
    int i;
    for(i = 0; i < 10; i++)      /* 模拟 10 条指令 */
    {
        int *cmd = (int *)malloc(sizeof(int));
        if(cmd) { *cmd = i; ThreadQueueAPI_PutMsg(ctx->tq, cmd); }
        usleep(200 * 1000);      /* 5Hz 指令 */
    }
    return NULL;
}

/* ========== 指令执行线程：GetMsg 一条一条执行 ========== */
static void *cmd_exec_thread(void *arg)
{
    Ctx *ctx = (Ctx *)arg;
    while(1)
    {
        int *cmd = (int *)ThreadQueueAPI_GetMsg(ctx->tq, 100);   /* 100ms 超时 */
        if(cmd == NULL)
        {
            if(ThreadQueueAPI_IsClosed(ctx->tq) > 0) break;      /* 关闭且空 → 退出 */
            continue;
        }
        printf("[cmd] execute #%d\n", *cmd);   /* 执行指令 */
        free(cmd);
    }
    return NULL;
}

/* ========== main ========== */
int main(void)
{
    Ctx ctx = {0};
    pthread_t t_acq, t_dump, t_cprod, t_cexec;
    T_StreamBufferConfig sbcfg = { 65536, 16384 };   /* 容量64K, 阈值16K(攒批) */

    /* 初始化 StreamBuffer(原始写盘) 与 ThreadQueue(指令) */
    if(StreamBufferAPI_Init(&ctx.sb, &sbcfg, "rawbin") != 0) return -1;
    if(ThreadQueueAPI_InitMsg(&ctx.tq, 64, "cmd", cmd_release) != 0) return -1;

    pthread_create(&t_acq,   NULL, acq_thread,     &ctx);
    pthread_create(&t_dump,  NULL, dump_thread,    &ctx);
    pthread_create(&t_cprod, NULL, cmd_prod_thread,&ctx);
    pthread_create(&t_cexec, NULL, cmd_exec_thread,&ctx);

    /* 等采集结束 */
    pthread_join(t_acq, NULL);
    printf("[main] acquire done, closing...\n");

    /* 优雅关闭：StreamBuffer 排空写盘 + ThreadQueue 排空指令 */
    StreamBufferAPI_Close(ctx.sb);        /* 阻止新写入 + broadcast; dump 线程取空后退出 */
    pthread_join(t_dump, NULL);

    ThreadQueueAPI_CloseMsg(ctx.tq);      /* 指令队列关闭 */
    pthread_join(t_cprod, NULL);
    pthread_join(t_cexec, NULL);          /* 执行完剩余指令后退出 */

    StreamBufferAPI_Destroy(&ctx.sb);
    ThreadQueueAPI_DestroyMsg(&ctx.tq);
    printf("[main] all done\n");
    return 0;
}
```

**关闭顺序要点**：采集线程 join 后 → `StreamBuffer Close`（写盘线程排空 bin 后退出）→ `ThreadQueue Close`（指令执行线程跑完剩余指令后退出）→ Destroy。保证 bin 数据完整、指令不丢。

## 七、消耗估算

| 模块 | 操作 | 估算 CPU 占用（Cortex-A7 @ 500MHz+） |
|------|------|------------------------------------|
| 采集 PutData(12KB) | memcpy 12KB + 环形 | ~5μs/条 × 100Hz ≈ < 0.1% |
| 写盘 GetDataAddress+fwrite | 零拷贝取址 + 磁盘 I/O | I/O 为主，攒批后频率低 |
| 中值滤波 | 27×10 插入排序 | ~12 万次/s ≈ < 0.1% |
| 指令队列 | 偶发 | 可忽略 |
| **合计（不含磁盘 I/O 等待）** | — | **< 1% CPU** |

> 主要瓶颈是**磁盘写速率**（1.2MB/s 持续写），需确保存储（eMMC/SD）跟上；若跟不上，StreamBuffer 会满则丢新（丢最新原始帧，但滤波/指令不受影响）。

## 八、扩展：何时改用 WindowQueue

本方案中值滤波用**直接数组**（单线程独占窗口，最省）。**改用 `WindowQueue`** 的情形：

- 窗口需要被**多线程并发读**——例如滤波线程填窗口，同时还有"上报线程"要把当前 10×27 窗口发出去。
- 想要库提供的**线程安全窗口访问**（`ForEach`/`Snapshot`）+ 满丢最老 + 运行统计。

```c
/* WindowQueue 版滤波窗口（容量 10，元素=27 关键点结构体） */
typedef struct { int v[NCH]; } KeyPts;
T_WindowQueueMsg *wq;
WindowQueueAPI_Init(&wq, WIN, sizeof(KeyPts), "kpwin", NULL);

/* 每来一条：Put(满丢最老) → ForEach 取窗口 → 27 通道排序中值 */
KeyPts kp = { ... };
WindowQueueAPI_Put(wq, &kp);
double scratch[WIN];
WindowQueueAPI_ForEach(wq, collect_cb, scratch);   /* 累积窗口(见 WindowQueue readme) */
/* 对 scratch 排序取中值... */
```

代价：多一点锁/队列开销（但 10.8KB/s 数据量，开销依旧可忽略）。**单线程滤波无需上 WindowQueue，直接数组最省。**

```makefile
# 关键：同时链接 StreamBuffer 与 ThreadQueue 静态库 + pthread
CFLAGS = -g -Wall -Wextra -I StreamBuffer/include -I ThreadQueue/include

         ThreadQueue/debug/libThreadQueue.a \

	arm-linux-gnueabihf-gcc $(CFLAGS) -o sensor_demo sensor_demo.c $(LDLIBS)
```

（先用 `make install_lib` 把两库装到公共路径，或直接用相对路径引用各自 `debug/*.a`。）

## 十、小结

| 需求 | 方案 | 库/实现 |
|------|------|--------|
| 原始 12KB → bin | 攒批 + 零拷贝写盘 | **StreamBuffer** (`GetDataAddress`) |
| 27 关键点滑窗中值滤波 | 环形数组 + 插入排序 | **直接数组（最省）** / 可选 WindowQueue |
| 控制指令一条条执行 | FIFO、满则阻塞不丢 | **ThreadQueue** |

三个库 + 一个轻量滤波函数，即可搭建完整的 100Hz 采集→记录→滤波→指令系统，CPU 占用极低，且采集线程永不被写盘/处理拖慢（生产不阻塞）。
