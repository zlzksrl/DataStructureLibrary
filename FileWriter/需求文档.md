# 需求文档 — FileWriter 文件写入模块（异步日志/CSV/二进制记录）

> **定位**：基于 StreamBuffer + ThreadManage 的异步文件写入库。提供 printf 式格式化接口，攒批写盘（减磁盘磨损），支持日志/CSV/二进制，自动文件轮转（按数量+时间命名），优雅关闭。
> **平台**：IMX6ULL (ARM Linux) | **作者**：zlzksrl | **草案**：2026-07-10
> **状态**：需求完善中（设计决策见第五节）

---

## 一、原始需求（用户提供）

1. 实现一个嵌入式代码用于记录日志、CSV 表格、二进制数据的库。
2. 使用 StreamBuffer 库 + ThreadManage 库实现。
3. 创建目录/多级路径/目标文件；获取当前写文件的不带路径的名字、绝对路径（含名字）、绝对路径（不含名字）。
4. 一定规则下同一类型创建的文件（如 `类型1_序号_时间.log`），可设最大文件数量，超过则删最老；也可无限制。
5. 获取当前写文件目录下的文件数量。
6. 写日志时可选每条数据前组合时间戳。
7. 写磁盘要收集一段数据后批量写入（减少频繁写操作）。
8. 写日志接口要有格式化输入功能（printf 式）。
9. 一个进程可能写多个文件，模块会被多处初始化多次，须**可重入**。
10. 停止记录时**优雅关闭**：阻止写入 → 缓存全刷盘 → 关闭文件 → 注销消费线程。
11. CSV 表头/各行内容由外部组织格式，库只管写（类似 printf 输出到文件）。

---

## 二、需求理解：异步文件写入管道

```
业务线程(多个)                         FileWriter 消费线程(每个实例1个)
   │ FileWriter_Write(fmt, ...)          │
   │  → vsnprintf 格式化                  │
   │  → [可选] 加时间戳前缀                │
   │  → StreamBuffer.PutData() 入队       │
   ▼                                     ▼
              [StreamBuffer 攒批缓冲]
                     │  used≥阈值 / 超时 / 关闭
                     ▼
              FileWriter 消费线程
              Wait → GetData → fwrite → fflush
                     │
                     ▼
              磁盘文件（自动轮转）
```

- **生产者（业务线程）**：调 `Write(fmt, ...)` → 格式化 → 入 StreamBuffer，不阻塞、不等磁盘。
- **消费者（FileWriter 内置线程）**：等触发 → 批量取 → fwrite + fflush。
- **解耦**：业务不被磁盘 I/O 拖慢；攒批减少 fwrite 调用次数。

## 三、依赖库选择

| 库 | 角色 | 说明 |
|----|------|------|
| **StreamBuffer** | 攒批缓冲 | PutData 写入 → Wait/GetData 批量取出 fwrite（主要方案） |
| **ThreadManage** | 消费线程创建 | 用 ThreadAPI 创建消费线程，SCHED_RR 轮转策略，可配优先级 |

> FileWriter 依赖两个库：StreamBuffer（攒批）+ ThreadManage（线程管理）。
>
> **注**：早期草案曾计划用 MemoryPool 做格式化 buffer 池，实际实现评估后精简为**栈 buffer**（`char buf[FW_FORMAT_BUF_SIZE]`），理由：
> - Write 内 Alloc→vsnprintf→PutData(拷贝入 SB)→Free 全程同步完成，buffer 生命周期极短，无跨线程共享需求；
> - StreamBuffer 是拷贝式入队，PutData 返回后 buf 立即可复用/释放，栈变量完全等价；
> - 少一层依赖，减少 mutex 开销和代码复杂度。
> - 未来若扩展"结构化异步 API"（业务线程 Alloc buffer→塞给消费线程→线程消费后归还），再引入 MemoryPool。

## 四、功能规格

### 4.1 初始化 `FileWriter_Init`
- 入参：配置结构体（见下）+ 实例句柄二级指针。
- 功能：
  - 创建目录/多级路径（`mkdir -p` 语义）。
  - 按命名规则创建第一个文件。
  - 初始化 StreamBuffer（容量/阈值/超时）。
  - 用 **ThreadManage ThreadAPI** 创建消费线程（SCHED_RR + 配置优先级）。
- **可重入**：每次 Init 创建独立实例（独立 StreamBuffer、独立文件句柄、独立线程），互不干扰。

### 4.2 配置结构体 `T_FileWriterConfig`
```c
typedef struct {
    /* 路径与命名 */
    char     dir_path[256];     /* 写入根目录（绝对或相对），自动创建多级 */
    char     date_subdir_prefix[16]; /* 日期子目录前缀，如 "X" → /log/X20260711/；空串=不按日期分目录 */
    char     file_prefix[64];   /* 文件名前缀，如 "sensor1" */
    char     file_ext[16];      /* 扩展名，如 ".log" / ".csv" / ".bin" */
    int      max_files;         /* 最大文件数量，0=无限制；>0 则超过删最老 */
    int      max_file_size;     /* 单文件最大字节，0=不限制；>0 则达此大小自动轮转。1G=1024*1024*1024 */
    int      auto_rotate_daily; /* 1=跨日自动轮转（建新日期目录+新文件）；0=不自动 */
    /* 线程 */
    int      thread_priority;  /* 消费线程 SCHED_RR 优先级（1~99，如 20）；0=用默认 */
    /* 写入行为 */
    int      timestamp;         /* 1=每行前加时间戳 "[2026-07-10 12:00:00] " */
    int      flush_bytes;       /* 攒批字节阈值（如 4096），达此值写盘 */
    int      flush_ms;          /* 定时写盘周期（如 100ms） */
    int      buffer_capacity;   /* StreamBuffer 容量（如 65536，须 2 的幂） */
} T_FileWriterConfig;
```

### 4.3 写入接口 `FileWriter_Write` / `FileWriter_WriteBin`
- **格式化写入**（printf 式）：
  ```c
  int FileWriter_Write(T_FileWriter *fw, const char *fmt, ...);
  ```
  - 内部 `vsnprintf` 格式化到栈 buffer（`FW_FORMAT_BUF_SIZE`=1KB，单条超长自动截断）。
  - [可选] 前置时间戳（`timestamp==1` 时），格式 `[HH:MM:SS.mmmmmm] `。
  - 组合后 `StreamBuffer_PutData()` 入队（满则丢新，不阻塞业务）。
  - PutData 内部 memcpy 到环形缓冲，Write 返回后栈 buffer 立即回收。
- **二进制写入**：
  ```c
  int FileWriter_WriteBin(T_FileWriter *fw, const void *data, int len);
  ```
  - 直接 PutData 原始字节（不加时间戳、不格式化）。

### 4.4 目录结构与文件命名

#### 4.4.1 路径组装与目录创建

**路径组装顺序**（从外到内）：
```
{dir_path}
  / {date_subdir_prefix}{YYYY_MM_DD}   ← date_subdir_prefix 非空时才有此层
  / {file_prefix 的路径部分}            ← file_prefix 含 "/" 时才有子目录
  / {file_prefix 的文件名部分}{序号3位}_{YYYY-MM-DD-HH-MM-SS}.{ext}
```

**file_prefix 含路径分隔的示例**：
- `file_prefix = "sensor1"` → 文件直接在日期目录下：
  ```
  /log/X2026_07_11/sensor1_000_2026-07-11-12-41-30.log
  ```
- `file_prefix = "sensor/sensor1"` → 在日期目录下再建 `sensor/` 子目录：
  ```
  /log/X2026_07_11/sensor/sensor1_000_2026-07-11-12-41-30.csv
  ```

**目录创建逻辑**（`mkdir -p` 语义）：
1. 先组装完整绝对路径（拆分为目录部分 + 文件名部分）。
2. 逐级检查路径每一层是否存在（`access`/`stat`）。
3. 不存在的层依次 `mkdir`（从外到内）。
4. 最后 `fopen` 创建文件。

> 无论 `date_subdir_prefix` 是否为空、`file_prefix` 是否含 `/`，都走同一个"组装完整路径 → 逐级检查 → 逐级创建"流程，统一处理。

#### 4.4.2 文件命名规则
- 格式：`{prefix}{序号3位}_{YYYY-MM-DD-HH-MM-SS}.{ext}`
  - 例：`sensor1_000_2026-07-11-12-41-30.csv`
  - 序号 3 位零填充（000, 001, ..., 999）。
- 完整路径示例（日期子目录开启）：
  ```
  /log/X20260711/sensor1_001_20260711_120000.log
  ```

#### 4.4.3 文件轮转
- 序号递增：每次创建新文件序号 +1。
- 最大数量限制：`max_files > 0` 时，创建新文件前检查当前写目录下**同文件名前缀**的文件数（匹配 `{file_prefix的文件名部分}_*`），超过则删最老的（按文件名中的日期时间排序）。
  - 例：`file_prefix="sensor/sensor1"`，目录 `/log/X2026_07_11/sensor/` 下统计 `sensor1_*.csv` 的数量。
  - 不同前缀的文件（如 `other_*.csv`）**不计入**。
- `max_files == 0`：无限制。
- **触发轮转的场景**：
  - Init 时创建第一个文件；
  - 手动 `Rotate()` 创建新文件；
  - 后续可扩展：文件大小达上限自动轮转、定时轮转。

### 4.5 优雅关闭 `FileWriter_Close`
```
FileWriter_Close()
   → StreamBuffer.Close（阻止新写入）
   → 消费线程排空（Wait→GetData→fwrite 直到 CLOSE_EMPTY）
   → fflush + fclose
   → 线程退出
   → 外部 join 或内部自管理
```

### 4.6 查询接口
| 函数 | 说明 |
|------|------|
| `GetCurrentFileName` | 当前写文件不带路径的名字（如 `sensor1_000_2026-07-11-12-41-30.csv`） |
| `GetCurrentFilePath` | 当前写文件绝对路径（含名字，如 `/log/X2026_07_11/sensor1_000_2026-07-11-12-41-30.csv`） |
| `GetCurrentDirPath` | 当前写文件绝对路径（不含名字，如 `/log/X2026_07_11`） |
| `GetFileCount` | 当前日期目录下同前缀的文件数量 |

### 4.7 销毁 `FileWriter_Destroy`
- Close + join 线程 + Destroy StreamBuffer + 释放资源。

## 五、关键设计决策（待确认）

### D1. 缓冲方案
- **StreamBuffer** ✓：字节流攒批，PutData(格式化后的字符串) → GetData → fwrite。

### D2. 消费线程
- **库内置消费线程** ✓：用 **ThreadManage 的 ThreadAPI** 创建（非裸 pthread_create）。
- **SCHED_RR 轮转策略**：消费线程使用 `SCHED_RR` 实时调度策略（不同进程中写盘线程优先级可能不同，需精确控制）。
- **可配优先级**：Init 时通过 `thread_priority` 字段配置（1~99，SCHED_RR 优先级范围）。
  - 典型值：日志/非关键文件写盘线程可设较低优先级（如 10~30），不抢占业务线程。
  - 不同进程的 FileWriter 实例可设不同优先级。

### D3. 文件轮转触发（初始化配置）
- **文件最大大小**：`max_file_size` 配置（字节），0=不限制；>0 时文件达到该大小自动轮转创建新文件。
- **跨日期自动轮转**：`auto_rotate_daily` 开关，1=跨日自动新建文件（配合日期子目录）。
- **手动 Rotate()**：随时可手动触发。
- 后续可扩展：定时轮转。

### D3.5 日期子目录
- `date_subdir_prefix` **非空**时按日期创建子目录（`{base}/{prefix}{YYYYMMDD}/`）。
- 消费线程每条写之前检查日期是否变化，变化则自动切目录+新建文件（跨日无缝）。
- `date_subdir_prefix` **为空串**时不分日期目录，所有文件直接在 `base_dir` 下。

### D4. 格式化 buffer 来源
- **栈 buffer** ✓（`char buf[FW_FORMAT_BUF_SIZE]`，1KB）：Write 函数栈上分配，vsnprintf 格式化后 memcpy 入 StreamBuffer，Write 返回时栈自动回收。
- 早期草案曾用 MemoryPool，评估后发现 Alloc/Free 全程同步且 buffer 无跨线程共享，池化无实际收益，遂改用栈变量。详见第三节"依赖库选择"注释。
- 单条日志 >1KB 会被截断（vsnprintf 返回值检查），不越界。

### D5. 时间戳格式
- **`[HH:MM:SS.mmmmmm] `**（含微秒，不含年月日——文件名已有日期）。
- 示例：`[16:50:44.789550] `
- 由 `timestamp` 开关控制（1=加前缀，0=不加）。

### D6. 库名 / 前缀
- **FileWriter**（`FileWriterAPI_*`）✓

## 六、公开 API 草案

```c
typedef struct T_FILEWRITER T_FileWriter;

int  FileWriterAPI_Init(T_FileWriter **fw, const T_FileWriterConfig *cfg);
int  FileWriterAPI_Destroy(T_FileWriter **fw);

int  FileWriterAPI_Write(T_FileWriter *fw, const char *fmt, ...);    /* printf 式格式化 */
int  FileWriterAPI_WriteBin(T_FileWriter *fw, const void *data, int len);  /* 二进制 */

int  FileWriterAPI_Flush(T_FileWriter *fw);     /* 立即触发一次写盘 */
int  FileWriterAPI_Close(T_FileWriter *fw);      /* 优雅关闭 */
int  FileWriterAPI_Rotate(T_FileWriter *fw);     /* 手动轮转（创建新文件） */

/* 查询 */
int  FileWriterAPI_GetCurrentFileName(T_FileWriter *fw, char *out, int out_len);
int  FileWriterAPI_GetCurrentFilePath(T_FileWriter *fw, char *out, int out_len);
int  FileWriterAPI_GetCurrentDirPath(T_FileWriter *fw, char *out, int out_len);
int  FileWriterAPI_GetFileCount(T_FileWriter *fw);
```

## 七、典型应用

- **异步日志**：`FileWriter_Write(fw, "[moduleA] ret=%d\n", ret);` → 自动加时间戳 → 攒批写盘。
- **CSV 记录**：外部组织行内容 → `FileWriter_Write(fw, "%d,%d,%.3f\n", ts, ch, val);`。
- **二进制 bin**：`FileWriter_WriteBin(fw, &frame, sizeof(frame));` → 攒批写盘。
- **多实例**：一个进程 Init 多个 FileWriter（不同目录/前缀），各自独立线程写盘。
