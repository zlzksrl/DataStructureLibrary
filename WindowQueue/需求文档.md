# 需求文档 — WindowQueue 滑动窗口队列

> 本文档为 **WindowQueue 独立数据结构库**的需求规格（隶属 `DataStructureLibrary` 仓库）。
> 目标平台：IMX6ULL (ARM Linux) | 作者：zlzksrl | 草案日期：2026-07-09
> 参考实现风格：`ThreadManage` 库
> 状态：**需求已定稿**（设计决策见第七节，公开 API 见第九节）

---

## 一、原始需求（用户提供）

1. 不间断采集传感器数据，数据为结构体，可能很大。
2. 需要一个队列，初始化时设置长度，将传感器数据放入队列。
3. 队列满时丢掉最老的数据。
4. 增加接口/回调函数，处理队列中**现在的数据（窗口）**，例如进行中值滤波运算。
5. 中途队列的长度可能修改（变长或变短）。
6. （预留）

---

## 二、需求理解与场景

### 2.1 应用场景
- 一个**采集线程**以固定/可变速率不断把传感器采样（结构体）写入队列。
- 一个或多个**处理线程**周期性地读取队列中**最近 N 条**数据作为一个**滑动窗口**，做信号处理：移动平均、中值滤波、Savitzky-Golay 滤波等。

### 2.2 为什么需要这个数据结构
| 已有结构 | 满时行为 | 数据读取方式 | 动态长度 | 是否满足本需求 |
|---------|---------|-------------|---------|--------------|
| **ThreadQueue**（环形 FIFO） | **阻塞**生产者 | **消费式** Get（取出即消失） | 不支持 | ✗ 满时阻塞采集；无法整体只读窗口 |
| **LatestQueue** | 覆盖（只保留**最新 1 条**） | 取最新 1 条 | 不支持 | ✗ 只留 1 条，无法做窗口滤波 |
| **WindowQueue（本次新增）** | **丢弃最老**，保留最近 N 条 | **只读遍历整个窗口** | **支持动态变长/变短** | ✓ |

**结论**：实现独立数据结构 **WindowQueue**——一个**有界、满则覆盖最老、支持整体只读访问、容量可动态调整**的滑动窗口缓冲区。

## 三、功能规格

### 3.1 队列定位
- **有界环形缓冲区**，容量 `size` 在 Init 时指定；按 `element_size`（结构体大小）**预分配连续内存** `size × element_size`。
- 始终保留**最近**的若干条数据（最多 `size` 条），按写入时间 FIFO 排列。
- 窗口内数据**时间序**：索引 [0] = 最老，[n-1] = 最新（n 为当前条数）。

### 3.2 写入数据 `WindowQueueAPI_Put(q, data)`
- `data` 为 `const void*`，指向用户待写入的结构体；**Put 内部按 `element_size` 拷贝**进环形缓冲尾部（成为最新）。
- **永不阻塞采集线程**：
  - 未满：直接拷贝写入，`nData++`。
  - 已满：**丢弃最老的一条**（直接覆盖，无需释放），再拷贝写入新的。
- 返回值 = 本次因满而丢弃的条数（`0` 未丢 / `1` 丢弃了最老 1 条），便于上层统计丢包率。
- 统计：每次成功 Put `ulTotalPut++`；若丢弃则 `ulTotalDiscarded++`，并按需更新峰值长度。

### 3.3 窗口数据访问（**核心**，滤波用）
提供**只读、不消费**的窗口访问，读后队列状态不变。两种形式：

1. **快照式 `WindowQueueAPI_Snapshot(q, out_buf, max_count)`**（推荐用于中值滤波等需排序/随机访问）
   - 拷贝 **最新的 `min(max_count, nData)` 条**：当窗口数据多于 `max_count` 时，取最近的 `max_count` 条；拷贝到 `out_buf` 后按 **老→新** 顺序连续排列。
   - `out_buf` 需至少 `返回值 × element_size` 字节。
   - 在锁内完成 memcpy 后**立即释放锁**，用户在锁外对数组排序/求和。**零内部 malloc**，最高效。
   - 返回实际拷贝条数 `n`（`0 ≤ n ≤ max_count`）。

2. **回调式 `WindowQueueAPI_ForEach(q, callback, user_ctx)`**（推荐用于移动平均等顺序聚合）
   - 回调签名：`void (*cb)(const void *data, int index, void *user_ctx)`，按 老→新 依次调用，`data` 指向队列内部该条数据，`index` 为窗口内序号（0 起）。
   - **回调在锁内执行**（`data` 直接指向环形缓冲条目，零拷贝零 malloc）；因此**回调必须快速返回**，否则阻塞采集线程的 Put。
   - ⚠️ 回调内**禁止**调用本队列任何 API（`mux` 非递归，会死锁）。

> 选型：`Snapshot` 适合大数据/慢算法（拷贝快、锁外处理）；`ForEach` 适合快速聚合（锁内、零拷贝）。

### 3.4 动态调整容量 `WindowQueueAPI_Resize(q, new_size)`
- `new_size > 0`。
- **变大**：重新分配更大缓冲区，按 FIFO 顺序搬移现有数据，全部保留。
- **变小**：从**最老端**丢弃 `(nData - new_size)` 条多余数据（直接覆盖，无需释放），其余搬移保留。
- 线程安全（锁内 realloc + 原子替换 buffer 指针 + 重置 lget/lput/nData/size）。

### 3.5 取出数据
- **不提供**消费式 `Get` / `Peek`（决策 D3）。处理线程通过 `Snapshot`/`ForEach` 只读访问，队列持续积累。
- 唯一会减少数据条数的操作：`Resize` 变小、`Flush`、`Destroy`。

### 3.6 生命周期管理
- `WindowQueueAPI_Init(&q, iQueueLen, iElementSize, sQueueName)`：预分配 `iQueueLen × iElementSize` 内存。
- `WindowQueueAPI_Close(q)`：关闭，Put 返回 -2（冻结窗口，供最后只读）。
- `WindowQueueAPI_Reopen(q)`：重新打开（前置条件：已关闭）。本结构无阻塞线程，故无“等待线程”约束。
- `WindowQueueAPI_IsClosed(q)`：查询关闭状态。
- `WindowQueueAPI_GetLength(q)`：当前窗口数据条数 `n`（`0 ≤ n ≤ size`）。
- `WindowQueueAPI_GetCapacity(q)`：容量 `size`。
- `WindowQueueAPI_Flush(q, callback, user_ctx)`：销毁前把窗口数据逐条（老→新）经回调处理后**清空**队列。
- `WindowQueueAPI_Destroy(&q)`：释放预分配内存与结构体，置 NULL。

### 3.7 运行统计 `WindowQueueAPI_StatsGet(q, pt_Stats)`（参照 ThreadManage `ThreadPoolStatsGet`）
- 通过 `T_WindowQueueStats` 返回：
  - `ulTotalPut`：累计成功 Put 次数；
  - `ulTotalDiscarded`：累计因满丢弃的最老数据条数（丢包率 = `ulTotalDiscarded / ulTotalPut`）；
  - `iPeakLength`：窗口历史峰值条数。
- 用于高频采集场景的丢包率与负载监控。

### 3.8 入队回调（Push 轻量处理，可选）
- `WindowQueueAPI_SetPutCallback(q, cb, ctx)` 注册后，**每次成功 Put 在锁内**构建当前窗口的只读指针视图并调用 `cb`。
- **零拷贝**：`entries[i]` 直接指向队列内部第 i 条数据（老→新），`count` 为当前条数。
- **未注册时零额外开销**：不注册回调，Put 与无回调版完全一致；注册后仅多 O(count) 次指针填充（无数据拷贝、无 malloc）。
- 适用：每条数据入队即做**轻量即时处理**——累加、均值、阈值判断、计数等（O(n) 简单运算）。
- ⚠️ 约束（锁内，与 ForEach 同）：① 回调必须快速返回（勿做排序等重运算）；② 禁调本队列任何 API（死锁）；③ `entries` 仅回调期间有效（返回后可能被后续 Put 覆盖）。
- 与 `ForEach`（Pull，处理线程按周期，适合重滤波）的区别：入队回调是 Push，每次入队自动触发，适合即时轻量统计；两者可共存。

---

## 四、线程模型
- **采集线程**：高频 `Put`，永不阻塞（最坏丢弃 1 条最老数据）。
- **处理线程**：周期性 `Snapshot` / `ForEach` 做滤波，读后窗口不变。
- 多采集 / 多处理线程均安全（`pthread_mutex_t` 保护）。
- **不引入**条件变量（无阻塞等待需求）。

---

## 五、内存管理（值拷贝模式）
- 队列**持有数据副本**：Init 时一次性预分配 `size × element_size` 连续内存。
- **无 `release_callback`**：满则丢弃 / Resize 变小 / Flush 时，旧数据直接被覆盖，无需释放。
- 用户传入的 `data` 仅作拷贝源，Put 返回后用户可立即复用/释放该缓冲。
- `Destroy` 整体 `free` 预分配内存与结构体。

### 5.1 拷贝开销分析（性能保证）
- **丢老数据：零拷贝**。环形缓冲区丢老 = 读指针 `lget` 前进一格，O(1)，与 `element_size` 无关；保留的数据完全不搬移（区别于线性数组的整体前移）。
- **Put 写新：1 次 `memcpy(element_size)`**。值拷贝固有成本；ARM Cortex-A7 上几 KB 结构体为 μs 级，经评估可接受（决策：**保持值拷贝**，不引入零拷贝写入接口）。
- **Snapshot 输出：memcpy 窗口→用户缓冲**（用户主动取数据时发生）。如需避免输出拷贝，改用 **`ForEach`（锁内零拷贝）**。

## 六、错误码约定（与 ThreadManage 一致）
| 错误码 | 含义 |
|--------|------|
| `0`  | 成功（Put 未丢弃） |
| `-1` | 参数无效或未初始化 |
| `-2` | 队列已关闭（仅 Put 返回） |
| `>0` | 仅 Put：本次丢弃的最老数据条数（1） |

## 九、公开 API（定稿）

```c
/** WindowQueue 队列句柄前向声明（隐藏内部实现） */
typedef struct T_WINDOWQUEUEMSG T_WindowQueueMsg;

/** ForEach / Flush 回调原型：data 指向一条 element_size 字节的数据，index 为窗口内序号(0=最老) */
typedef void (*WindowQueueDataCb)(const void *data, int index, void *user_ctx);

/* ---- 生命周期 ---- */
int  WindowQueueAPI_Init(T_WindowQueueMsg **ppt_QueueMsg,
                         int iQueueLen, int iElementSize, const char *sQueueName);
int  WindowQueueAPI_Destroy(T_WindowQueueMsg **ppt_QueueMsg);
int  WindowQueueAPI_Close(T_WindowQueueMsg *pt_QueueMsg);
int  WindowQueueAPI_Reopen(T_WindowQueueMsg *pt_QueueMsg);
int  WindowQueueAPI_IsClosed(T_WindowQueueMsg *pt_QueueMsg);

/* ---- 写入与窗口访问 ---- */
int  WindowQueueAPI_Put(T_WindowQueueMsg *pt_QueueMsg, const void *data);
int  WindowQueueAPI_Snapshot(T_WindowQueueMsg *pt_QueueMsg,
                             void *out_buf, int max_count);   /* 拷贝最新 max_count 条，老→新排列 */
int  WindowQueueAPI_ForEach(T_WindowQueueMsg *pt_QueueMsg,
                            WindowQueueDataCb callback, void *user_ctx);

/* ---- 容量与查询 ---- */
int  WindowQueueAPI_Resize(T_WindowQueueMsg *pt_QueueMsg, int new_size);
int  WindowQueueAPI_GetLength(T_WindowQueueMsg *pt_QueueMsg);    /* 当前条数 n  */
int  WindowQueueAPI_GetCapacity(T_WindowQueueMsg *pt_QueueMsg);  /* 容量 size   */

/* ---- 刷新（销毁前处理窗口并清空） ---- */
int  WindowQueueAPI_Flush(T_WindowQueueMsg *pt_QueueMsg,
                          WindowQueueDataCb callback, void *user_ctx);

/* ---- 运行统计 ---- */
typedef struct T_WINDOWQUEUESTATS
{
    unsigned long ulTotalPut;        /**< 累计成功 Put 次数 */
    unsigned long ulTotalDiscarded;  /**< 累计因满丢弃的最老数据条数 */
    int           iPeakLength;       /**< 窗口历史峰值条数 */
} T_WindowQueueStats;
int  WindowQueueAPI_StatsGet(T_WindowQueueMsg *pt_QueueMsg, T_WindowQueueStats *pt_Stats);

/* ---- 入队回调（Push 轻量处理，锁内零拷贝） ---- */
typedef void (*WindowQueuePutCb)(const void * const *entries, int count, void *user_ctx);
int  WindowQueueAPI_SetPutCallback(T_WindowQueueMsg *pt_QueueMsg,
                                   WindowQueuePutCb cb, void *user_ctx);
```
